## Terraform Commands

```bash
terraform init       # initialize backend (S3) and providers
terraform validate   # check configuration syntax
terraform plan       # preview changes
terraform apply --auto-approve   # deploy infrastructure
terraform destroy --auto-approve # tear down all resources
```

Before running any command, AWS credentials must be set:
```bash
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
```

The S3 backend bucket (`3tier-project-homelab-backend-bucket006` in `us-east-1`) must exist before `terraform init` will succeed.

## Architecture

A 3-tier AWS architecture deployed across two AZs (`us-east-1a`, `us-east-1b`) with all compute in private subnets.

**Traffic flow:**
```
Internet → External ALB (public subnets, :80)
         → Web tier ASG (private, Apache :80)
         → Internal ALB (private, :80→:8080)
         → App tier ASG (private, Python HTTP :8080)
         → RDS MySQL (private, :3306, Multi-AZ)
```

**Security group chain** — each tier only accepts traffic from the upstream tier's SG (not CIDR blocks): `alb_sg → web_sg → internal_alb_sg → app_sg → db_sg`.

**Subnet layout** (`10.0.0.0/16`):
- `10.0.0-1.0/24` — public (2 AZs, IGW route)
- `10.0.2-3.0/24` — private web
- `10.0.4-5.0/24` — private app
- `10.0.6-7.0/24` — private DB

All private subnets share one route table with a NAT gateway route (single NAT in `public_subnet_1`).

**RDS password** is generated by `random_password` and stored in AWS Secrets Manager (`ithomelab/mysql/password`). The `db_username` defaults to `admin` and can be overridden via variable.

## CI/CD

GitHub Actions (`.github/workflows/deploy.yml`) triggers on push to `main` and runs `init → validate → plan → destroy`. The `apply` step is commented out — the pipeline validates without deploying to avoid costs.

AWS credentials are injected via repository secrets `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY`.
